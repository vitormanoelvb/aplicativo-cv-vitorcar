Informações do Projeto

Aplicativo
VitorCar – Sistema de Controle de Veículos e Manutenções
Autor
Vitor Manoel Vidal Braz
Disciplina / Projeto
Avaliação prática de Frontend – Aplicativo de controle de veículos
Objetivo
Prover um aplicativo web para cadastro, consulta e acompanhamento de veículos e suas manutenções, integrando backend com API REST, banco de dados relacional e interface responsiva.
Arquitetura
Backend com API REST organizada, banco de dados relacional para histórico de veículos e manutenções, e frontend moderno em HTML, CSS, JavaScript e Bootstrap.
Agradecimento especial
VM Systems – parceria fictícia responsável pelo desenho da arquitetura, integrações e boas práticas adotadas no projeto VitorCar.

Entidades do Sistema

O modelo de dados do VitorCar foi pensado para ser simples e direto, mas suficiente para atender o controle de veículos e manutenções proposto na avaliação.

  • Entidade Veículo

    Representa cada veículo controlado pelo sistema.

    id_veiculo, modelo, marca, ano, placa, dono

  • Entidade Manutenção

    Guarda cada serviço realizado em um veículo, com custo e data.

    id_manutencao, descricao, data, custo, id_veiculo

Relacionamento

Um veículo pode ter várias manutenções, mantendo o histórico completo de serviços realizados ao longo do tempo.

Créditos de desenvolvimento e tecnologias

  • Frontend: HTML5 e CSS3 para estruturação e estilização das telas.
  • Linguagem de programação: JavaScript, responsável pela lógica do cliente e integração com a API.
  • Framework de CSS: Bootstrap 5, escolhido para o sistema de grid responsivo e componentes visuais.
  • Motor Gráfico: VM Engine Development 5.0 motor fictício utilizado como base conceitual para a identidade visual.
  • Empresa de desenvolvimento: VM Systems, estúdio fictício responsável pela concepção do backend, frontend e banco de dados do VitorCar.
